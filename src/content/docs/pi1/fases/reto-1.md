title: Reto 1 - Identificación del Problema
description: Define la idea, el público objetivo y valida el problema a resolver
---

## Objetivo

Identificar una necesidad real del mercado o de un colectivo específico y definir una solución software que la resuelva. Este reto sienta las bases de todo el proyecto.

## Entregables

### 1. Idea del Proyecto

Documento que describe:

- **Nombre del proyecto**: debe ser descriptivo y único
- **Problema identificado**: ¿qué necesidad existe? ¿cómo se detectó?
- **Solución propuesta**: ¿cómo resuelve el software ese problema?
- **Contexto de uso**: ¿dónde y cuándo se utilizaría?

### 2. Público Objetivo

Identificación clara de los usuarios finales:

- Perfil demográfico (edad, formación, sector)
- Necesidades específicas del usuario
- Tamaño estimado del mercado potencial
- Canales de acceso al usuario

### 3. Lean Canvas

Completar un Lean Canvas con los siguientes bloques:

| Bloque | Descripción |
|---|---|
| Problema | Top 3 problemas que resuelve |
| Segmento de clientes | Usuarios objetivo |
| Propuesta de valor única | ¿Qué te diferencia? |
| Solución | Producto mínimo viable |
| Canales | ¿Cómo llegarás a los usuarios? |
| Fuentes de ingresos | Modelo de negocio (si aplica) |
| Estructura de costes | Recursos necesarios |
| Métricas clave | Indicadores de éxito |
| Ventaja competitiva | ¿Qué te hace único? |

## Criterios de Evaluación

- Originalidad y viabilidad de la idea
- Claridad en la definición del problema
- Identificación precisa del público objetivo
- Lean Canvas completo y coherente
- Presentación estructurada del documento
