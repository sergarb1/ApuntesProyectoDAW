title: Reto 4 - Implementación del MVP
description: Desarrollo del producto mínimo viable con base de datos y aplicación funcional
---

## Objetivo

Construir un prototipo funcional de la aplicación. El MVP debe incluir las funcionalidades esenciales definidas en los retos anteriores y demostrar que la idea es técnicamente viable.

## Entregables

### 1. Base de Datos

- Script con la estructura completa de tablas
- Datos de prueba para demostrar funcionalidad
- Consultas SQL representativas de las operaciones del sistema

### 2. Aplicación Funcional

Desarrollo del MVP con:

- **Frontend básico**: HTML, CSS, JavaScript para las vistas principales
- **Backend funcional**: lógica de negocio implementada en el lenguaje del curso (Java, PHP, Python, Node.js)
- **Conexión a base de datos**: operaciones CRUD funcionales
- **Navegación**: flujo completo entre pantallas

### 3. Repositorio Git

El código debe gestionarse con control de versiones:

- Repositorio en GitHub (privado, con acceso al profesor)
- Commits frecuentes y mensajes descriptivos
- README.md con instrucciones de instalación y uso
- `.gitignore` adecuado al lenguaje y herramientas

### Requisitos Técnicos Mínimos

- La aplicación debe ejecutarse sin errores
- Debe existir al menos un flujo completo (registro → acción principal → resultado)
- El código debe estar organizado en carpetas con separación de responsabilidades
- La interfaz debe ser usable aunque no tenga diseño profesional

## Criterios de Evaluación

- Funcionalidad básica completa y sin errores
- Conexión a base de datos operativa
- Estructura del código limpia y organizada
- Uso correcto de Git (commits, ramas, README)
- El MVP cumple con lo especificado en los requisitos
