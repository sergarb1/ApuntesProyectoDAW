## Sistema de Evaluación

La evaluación del módulo de Proyecto Intermodular I es **continua, formativa e integradora**. Se basa en la superación de los resultados de aprendizaje mediante la entrega de los distintos retos.

### Instrumentos de Calificación

| Instrumento | Descripción | Peso | Nota Mínima |
|---|---|---|---|
| Documentación | Retos 1, 2 y 3: idea, análisis y diseño | 30% | >= 5 |
| Proyecto | Reto 4: implementación del MVP | 40% | >= 5 |
| Presentación | Reto 5: demo, memoria y vídeo | 20% | >= 5 |
| Participación | Intervenciones en clase y foros, coevaluación | 10% | No |

### Requisitos para Aprobar

1. Obtener una calificación mínima de **5.00 sobre 10** en cada instrumento (excepto participación)
2. La nota final se calcula como la media ponderada de los instrumentos superados
3. Si algún instrumento no se supera, la calificación máxima será 4.9

### Criterios de Evaluación por Reto

#### Reto 1 (Identificación)
- CE1.1: Identifica una necesidad real del sector
- CE1.2: Define el público objetivo con precisión
- CE1.3: Completa el Lean Canvas correctamente

#### Reto 2 (Análisis)
- CE2.1: Redacta historias de usuario con criterios de aceptación
- CE2.2: Identifica actores y casos de uso del sistema
- CE2.3: Documenta los flujos principal y alternativos

#### Reto 3 (Diseño)
- CE3.1: Diseña el diagrama de clases del dominio
- CE3.2: Crea el modelo E/R normalizado
- CE3.3: Elabora wireframes de las pantallas principales

#### Reto 4 (Implementación)
- CE4.1: Implementa el MVP con las funcionalidades básicas
- CE4.2: Conecta la aplicación con la base de datos
- CE4.3: Utiliza Git para el control de versiones

#### Reto 5 (Presentación)
- CE5.1: Realiza una demo funcional del proyecto
- CE5.2: Entrega la memoria técnica completa
- CE5.3: Defiende el proyecto ante el tribunal

### Convocatorias

- **Convocatoria ordinaria**: junio, tras la entrega del Reto 5
- **Convocatoria extraordinaria**: finales de junio, con los mismos instrumentos y criterios
