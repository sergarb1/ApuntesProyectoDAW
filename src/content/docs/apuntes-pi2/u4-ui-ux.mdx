## Introducción

El diseño de interfaces (UI) y la experiencia de usuario (UX) determinan cómo los usuarios interactúan con tu aplicación. Un buen diseño mejora la usabilidad, la accesibilidad y la satisfacción del usuario.

## Principios de diseño UX

### 1. Usabilidad
- **Consistencia**: mismos patrones en toda la interfaz
- **Simplicidad**: menos es más; elimina elementos innecesarios
- **Retroalimentación**: el usuario siempre sabe qué está pasando
- **Prevención de errores**: mejor prevenir que corregir

### 2. Jerarquía visual
Organiza los elementos por importancia usando:
- **Tamaño**: más grande = más importante
- **Color**: colores llamativos para acciones principales
- **Espaciado**: más espacio = mayor relevancia
- **Tipografía**: negritas para títulos, normal para cuerpo

### 3. Ley de Fitts
El tiempo para alcanzar un objetivo depende de su tamaño y distancia. Los botones principales deben ser grandes y estar cerca.

## Prototipado con Figma

Figma es una herramienta colaborativa de diseño de interfaces.

### Flujo de trabajo

1. **Wireframes**: estructura básica sin estilo (bocetos en gris)
2. **Mockups**: diseño visual completo con colores, tipografía, imágenes
3. **Prototipos**: mockups interactivos con enlaces entre pantallas

### Prácticas recomendadas en Figma

- Usa **auto layout** para diseños responsive
- Define **componentes reutilizables** (botones, inputs, cards)
- Crea **variantes** para estados (hover, active, disabled)
- Utiliza **estilos compartidos** (colores, tipografía, sombras)

## Tailwind CSS

Tailwind es un framework CSS utility-first que acelera el desarrollo.

```html
<!-- Sin Tailwind -->
<div style="background: white; border-radius: 8px; padding: 16px;
            box-shadow: 0 2px 8px rgba(0,0,0,0.1);">
    <h2 style="font-size: 1.25rem; font-weight: 600;">Título</h2>
    <p style="color: #666;">Descripción</p>
</div>

<!-- Con Tailwind -->
<div class="bg-white rounded-lg p-4 shadow-md">
    <h2 class="text-xl font-semibold">Título</h2>
    <p class="text-gray-500">Descripción</p>
</div>
```

### Instalación

```bash
npm install -D tailwindcss @tailwindcss/vite
```

```javascript
// vite.config.js
import { defineConfig } from 'vite';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
    plugins: [tailwindcss()]
});
```

```css
/* src/index.css */
@import "tailwindcss";
```

## Accesibilidad (WCAG)

Las Pautas de Accesibilidad al Contenido Web (WCAG) garantizan que tu web sea usable por todas las personas.

### Principios WCAG

| Principio | Significado |
|-----------|-------------|
| Perceptible | La información debe presentarse de forma que todos puedan percibirla |
| Operable | Todos los componentes de la interfaz deben ser operables |
| Comprensible | La información y la interfaz deben ser comprensibles |
| Robusto | El contenido debe ser interpretado por diferentes agentes de usuario |

### Buenas prácticas de accesibilidad

```html
<!-- 1. Contraste suficiente -->
<!-- ❌ Incorrecto: contraste insuficiente -->
<p style="color: #999;">Texto claro sobre fondo blanco</p>

<!-- ✅ Correcto: ratio de contraste ≥ 4.5:1 -->
<p class="text-gray-700">Texto legible</p>

<!-- 2. Texto alternativo en imágenes -->
<img src="grafico.jpg" alt="Evolución de ventas 2025-2026">

<!-- 3. Etiquetas en formularios -->
<label for="email">Email:</label>
<input id="email" type="email" aria-describedby="email-help">
<p id="email-help">Nunca compartiremos tu email</p>

<!-- 4. Roles ARIA -->
<nav aria-label="Navegación principal">
<ul role="menubar">
    <li role="menuitem"><a href="/">Inicio</a></li>
</ul>
</nav>

<!-- 5. Skip to content (saltar navegación) -->
<a href="#main-content" class="sr-only focus:not-sr-only">
    Saltar al contenido principal
</a>
<main id="main-content">
```

### Atajos de teclado y foco visible

```css
/* Foco visible para navegación por teclado */
a:focus-visible,
button:focus-visible,
input:focus-visible {
    outline: 2px solid #2563eb;
    outline-offset: 2px;
}

/* Ocultar visualmente pero mantener accesible */
.sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border-width: 0;
}
```

## Sistema de diseño

Un sistema de diseño unifica la apariencia de toda la aplicación.

```css
/* tokens.css */
:root {
    /* Colores */
    --color-primary: #2563eb;
    --color-primary-dark: #1d4ed8;
    --color-secondary: #7c3aed;
    --color-background: #ffffff;
    --color-text: #1a1a2e;
    --color-text-muted: #64748b;

    /* Tipografía */
    --font-sans: 'Inter', system-ui, sans-serif;
    --font-mono: 'JetBrains Mono', monospace;
    --text-xs: 0.75rem;
    --text-sm: 0.875rem;
    --text-base: 1rem;
    --text-lg: 1.125rem;
    --text-xl: 1.25rem;
    --text-2xl: 1.5rem;
    --text-3xl: 1.875rem;

    /* Espaciado */
    --space-1: 0.25rem;
    --space-2: 0.5rem;
    --space-4: 1rem;
    --space-6: 1.5rem;
    --space-8: 2rem;

    /* Bordes */
    --radius-sm: 4px;
    --radius-md: 8px;
    --radius-lg: 12px;

    /* Sombras */
    --shadow-sm: 0 1px 2px rgba(0,0,0,0.05);
    --shadow-md: 0 4px 6px rgba(0,0,0,0.1);
    --shadow-lg: 0 10px 15px rgba(0,0,0,0.1);
}
```

## Buenas prácticas

1. **Mobile first**: diseña para pantalla pequeña primero
2. **Consistencia visual**: mismos colores, tipografía y espaciado en toda la app
3. **Pruebas con usuarios reales**: observa cómo usan tu interfaz
4. **Accesibilidad desde el inicio**: es más fácil que adaptar después
5. **Iteración continua**: el diseño nunca está terminado; mejora basado en feedback
