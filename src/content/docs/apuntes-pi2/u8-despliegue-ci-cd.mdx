## Objetivos de la unidad

- Desplegar aplicaciones web en plataformas cloud (Railway, Vercel)
- Configurar pipelines CI/CD con GitHub Actions
- Gestionar variables de entorno en producción
- Configurar dominios personalizados y HTTPS

## Introducción

El despliegue es la fase final del desarrollo. Un pipeline CI/CD automatiza las pruebas y la publicación, garantizando que los cambios lleguen a producción de forma segura y rápida.

## Plataformas cloud

### Vercel (Frontend)

Vercel es la plataforma ideal para frontends estáticos y frameworks como React, Vue, Astro.

```bash
# Instalar CLI
npm i -g vercel

# Despliegue inicial
vercel

# Despliegue a producción
vercel --prod

# Variables de entorno
vercel env add SECRET_KEY
```


### Railway (Backend completo)

Railway despliega backends con BD incluidas.

```bash
# CLI
npm i -g @railway/cli

# Login
railway login

# Inicializar
railway init

# Desplegar
railway up

# Abrir en navegador
railway open
```

## GitHub Actions (CI/CD)

GitHub Actions automatiza build, test y deploy.

### Workflow básico: build + test

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'npm'

      - run: npm ci

      - run: npm run lint

      - run: npm test

      - run: npm run build
```

### Despliegue a Railway

```yaml
# .github/workflows/deploy-railway.yml
name: Deploy to Railway

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 22

      - name: Install Railway
        run: npm i -g @railway/cli

      - name: Deploy
        run: railway up
        env:
          RAILWAY_TOKEN: ${{ secrets.RAILWAY_TOKEN }}
```

### Despliegue a GitHub Pages

```yaml
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build
        run: npm run build
        env:
          PUBLIC_URL: https://${{ github.repository_owner }}.github.io/${{ github.event.repository.name }}

      - uses: actions/upload-pages-artifact@v3
        with:
          path: dist/

  deploy:
    runs-on: ubuntu-latest
    needs: build
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}

    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```

### Despliegue con Docker + SSH

```yaml
name: Deploy via Docker

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Build Docker image
        run: docker build -t mi-app:${{ github.sha }} .

      - name: Push to registry
        run: |
          docker tag mi-app:${{ github.sha }} ghcr.io/${{ github.repository }}:latest
          echo ${{ secrets.GITHUB_TOKEN }} | docker login ghcr.io -u ${{ github.actor }} --password-stdin
          docker push ghcr.io/${{ github.repository }}:latest

      - name: Deploy via SSH
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.SSH_HOST }}
          username: ${{ secrets.SSH_USER }}
          key: ${{ secrets.SSH_KEY }}
          script: |
            docker pull ghcr.io/${{ github.repository }}:latest
            docker stop mi-app || true
            docker rm mi-app || true
            docker run -d --name mi-app \
              -p 3000:3000 \
              --env-file /opt/mi-app/.env \
              ghcr.io/${{ github.repository }}:latest
```

## Variables de entorno en producción

## Buenas prácticas CI/CD

1. **Ejecutar tests antes del deploy**: nunca despliegues código que falle tests
2. **Variables de entorno en secrets**: nunca hardcodees credenciales
3. **Entornos por separado**: development → staging → production
4. **Rollback rápido**: poder volver a versión anterior fácilmente
5. **Notificaciones**: avisar al equipo cuando un deploy falla o se completa

## Estrategias de despliegue

### Blue-Green Deployment

```
┌──────────┐     ┌──────────┐
│  Blue    │     │  Green   │
│ (viejo)  │────>│ (nuevo)  │
└──────────┘     └──────────┘
   ↑ router cambia cuando Green está listo
```

### Rolling Update

```
Instancia 1: detener → actualizar → iniciar
Instancia 2: detener → actualizar → iniciar
Instancia 3: detener → actualizar → iniciar
(Sin downtime)
```

## Ejercicio práctico

Configura un pipeline completo para una app con:
- Tests automáticos en cada PR
- Build de Docker image
- Despliegue automático en main

<details>
<summary>Solución</summary>

```yaml
# .github/workflows/ci-cd.yml
name: CI/CD Pipeline

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  test:
    if: github.event_name == 'pull_request'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'npm'
      - run: npm ci
      - run: npm test
      - run: npm run lint

  build-and-deploy:
    if: github.event_name == 'push'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'npm'
      - run: npm ci
      - run: npm run build
        env:
          API_URL: ${{ secrets.API_URL }}
          PUBLIC_URL: https://miapp.com

      - name: Deploy to Vercel
        uses: amondnet/vercel-action@v25
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          vercel-args: '--prod'
```

</details>

## Resumen

- **Vercel**: ideal para frontend (React, Astro, Vue)
- **Railway**: despliegue completo incluyendo backend y BD
- **GitHub Actions**: CI/CD con workflows YAML
- **Variables de entorno**: secrets en GitHub, no hardcodeadas
- **Dominio personalizado**: CNAME + HTTPS automático
- **Estrategias**: blue-green, rolling update para zero downtime
