## Introducción

La seguridad web es crítica en cualquier aplicación. Las vulnerabilidades pueden exponer datos de usuarios, permitir accesos no autorizados o comprometer el servidor. Conocer y aplicar medidas preventivas es responsabilidad de todo desarrollador.

## OWASP Top 10 (2025)

OWASP publica periódicamente las 10 vulnerabilidades web más críticas:

| # | Vulnerabilidad | Descripción |
|---|---------------|-------------|
| 1 | Control de Acceso Roto | Usuarios acceden a recursos sin permisos |
| 2 | Fallos Criptográficos | Datos sensibles no cifrados |
| 3 | Inyección | SQL, NoSQL, OS command injection |
| 4 | Diseño Inseguro | Ausencia de controles de seguridad en el diseño |
| 5 | Configuración Incorrecta | Puertos abiertos, cabeceras inseguras |
| 6 | Componentes Vulnerables | Librerías desactualizadas |
| 7 | Fallos de Autenticación | Sesiones débiles, contraseñas sin hash |
| 8 | Integridad de Datos y Software | Actualizaciones no verificadas, CI/CD inseguro |
| 9 | Monitoreo y Logging Insuficientes | No detectar intrusiones |
| 10 | SSRF (Server-Side Request Forgery) | El servidor hace peticiones a destinos no previstos |

## Prevención de inyección SQL

```javascript
// ❌ VULNERABLE - concatenación directa
app.get('/api/usuario', (req, res) => {
    const id = req.query.id;
    const sql = `SELECT * FROM usuarios WHERE id = ${id}`;
    // Un atacante: ?id=1; DROP TABLE usuarios--
});

// ✅ SEGURO - prepared statements
app.get('/api/usuario', async (req, res) => {
    const [rows] = await pool.execute(
        'SELECT * FROM usuarios WHERE id = ?',
        [req.query.id]
    );
    res.json(rows);
});
```

### Validación y sanitización

```javascript
const { body, validationResult } = require('express-validator');

app.post('/api/comentarios',
    body('texto')
        .trim()                          // eliminar espacios
        .escape()                        // escapar HTML (< → &lt;)
        .isLength({ min: 1, max: 500 }), // longitud controlada
    (req, res) => {
        const errors = validationResult(req);
        if (!errors.isEmpty()) {
            return res.status(400).json({ errors: errors.array() });
        }
        // Almacenar comentario (ya sanitizado)
    }
);
```

## Cifrado de contraseñas

Nunca almacenes contraseñas en texto plano. Usa bcrypt (hash + salt).

```bash
npm install bcrypt
```

```javascript
const bcrypt = require('bcrypt');
const SALT_ROUNDS = 10;

// Registrar usuario
async function registrar(email, password) {
    const hash = await bcrypt.hash(password, SALT_ROUNDS);
    await db.execute(
        'INSERT INTO usuarios (email, password) VALUES (?, ?)',
        [email, hash]
    );
}

// Verificar login
async function login(email, password) {
    const [rows] = await db.execute(
        'SELECT * FROM usuarios WHERE email = ?',
        [email]
    );
    if (rows.length === 0) return null;

    const usuario = rows[0];
    const valido = await bcrypt.compare(password, usuario.password);

    return valido ? usuario : null;
}
```

### Otras medidas de autenticación

```javascript
// Política de contraseñas
function validarPassword(password) {
    const minLength = 8;
    const hasUpper = /[A-Z]/.test(password);
    const hasLower = /[a-z]/.test(password);
    const hasNumber = /\d/.test(password);
    const hasSpecial = /[!@#$%^&*]/.test(password);

    return password.length >= minLength
        && hasUpper && hasLower
        && hasNumber && hasSpecial;
}

// Rate limiting (evitar fuerza bruta)

app.post('/api/login', loginLimiter, loginController);
```

## Cabeceras de seguridad con Helmet

Helmet configura cabeceras HTTP de seguridad automáticamente.

```bash
npm install helmet
```

```javascript
const helmet = require('helmet');

app.use(helmet());

// Configuración personalizada
app.use(helmet({
    contentSecurityPolicy: {
        directives: {
            defaultSrc: ["'self'"],
            scriptSrc: ["'self'", "'unsafe-inline'"],
            styleSrc: ["'self'", "'unsafe-inline'"],
            imgSrc: ["'self'", "https:", "data:"],
            connectSrc: ["'self'", "https://api.mi-app.com"],
        },
    },
    // Forzar HTTPS
    hsts: {
        maxAge: 31536000,
        includeSubDomains: true,
        preload: true,
    },
}));
```

### Cabeceras que Helmet configura

| Cabecera | Protege contra |
|----------|---------------|
| `Content-Security-Policy` | XSS, inyección de scripts |
| `X-Content-Type-Options` | MIME sniffing |
| `X-Frame-Options` | Clickjacking |
| `Strict-Transport-Security` | Ataques MITM (HTTPS forzado) |
| `X-XSS-Protection` | XSS reflejado (legacy) |
| `Referrer-Policy` | Fugas de URL en referrer |

## CORS (Cross-Origin Resource Sharing)

CORS controla qué dominios pueden acceder a tu API.

```bash
npm install cors
```

```javascript
const cors = require('cors');

// ❌ Permitir todo (solo desarrollo)
app.use(cors());

// ✅ Configuración específica
app.use(cors({
    origin: [
        'https://miapp.com',
        'https://admin.miapp.com',
    ],
    methods: ['GET', 'POST', 'PUT', 'DELETE'],
    allowedHeaders: ['Content-Type', 'Authorization'],
    credentials: true,                 // cookies/tokens
    maxAge: 86400,                     // cachear preflight 24h
}));

// Para rutas específicas
app.get('/api/publica', cors(), (req, res) => {
    res.json({ data: 'accesible desde cualquier origen' });
});
```

## XSS (Cross-Site Scripting)

```javascript
// ❌ VULNERABLE - renderizar HTML directamente
function mostrarComentario(comentario) {
    elemento.innerHTML = comentario;
    // <script>alert('xss')</script> se ejecutaría
}

// ✅ SEGURO - usar textContent
function mostrarComentario(comentario) {
    elemento.textContent = comentario;
}

// En React/Vue: el framework escapa automáticamente
// ❌ Peligroso en React:
<div dangerouslySetInnerHTML={{ __html: comentario }} />
// ✅ Seguro (React escapa por defecto):
<div>{comentario}</div>
```

## Seguridad en JWT

```javascript
const jwt = require('jsonwebtoken');

const SECRET = process.env.JWT_SECRET;  // ← variable de entorno, nunca en código

// Token con información mínima
const token = jwt.sign(
    {
        id: usuario.id,
        rol: usuario.rol,
        // NO incluir: password, tarjetas, datos sensibles
    },
    SECRET,
    {
        expiresIn: '24h',
        issuer: 'mi-app',
        audience: 'mi-app-frontend',
    }
);

// Verificar
try {
    const decoded = jwt.verify(token, SECRET, {
        issuer: 'mi-app',
        audience: 'mi-app-frontend',
    });
    req.usuario = decoded;
} catch (err) {
    return res.status(403).json({ error: 'Token inválido o expirado' });
}
```

## Buenas prácticas generales

1. **Principio de mínimo privilegio**: cada usuario/rol solo accede a lo necesario
2. **Defense in depth**: múltiples capas de seguridad (no confíes en una sola medida)
3. **Validar todo**: nunca confíes en datos del cliente (frontend ni backend)
4. **Actualizar dependencias**: usa `npm audit` regularmente, Dependabot en GitHub
5. **Logging de seguridad**: registra intentos de acceso, errores de autenticación
6. **HTTPS siempre**: redirige HTTP a HTTPS, usa HSTS

## Ejercicio práctico

Audita y corrige las vulnerabilidades de este código:

```javascript
// ❌ ¿Qué vulnerabilidades tiene?
app.get('/api/usuario', (req, res) => {
    const id = req.query.id;
    const sql = "SELECT * FROM usuarios WHERE id = " + id;
    db.query(sql, (err, result) => {
        res.send(result);
    });
});
```

<details>
<summary>Solución</summary>

```javascript
// ✅ Versión segura
const { param, validationResult } = require('express-validator');
const helmet = require('helmet');
const cors = require('cors');
const rateLimit = require('express-rate-limit');

app.use(helmet());
app.use(cors({ origin: process.env.ALLOWED_ORIGINS?.split(',') }));

const limiter = rateLimit({
    windowMs: 15 * 60 * 1000,
    max: 100
});
app.use('/api/', limiter);

app.get('/api/usuario/:id',
    param('id').isInt().withMessage('ID debe ser numérico'),
    async (req, res) => {
        const errors = validationResult(req);
        if (!errors.isEmpty()) {
            return res.status(400).json({ errors: errors.array() });
        }

        try {
            const [rows] = await pool.execute(
                'SELECT id, nombre, email FROM usuarios WHERE id = ?',
                [req.params.id]
            );
            if (rows.length === 0) {
                return res.status(404).json({ error: 'No encontrado' });
            }
            res.json(rows[0]);
        } catch (error) {
            console.error('Error al buscar usuario:', error);
            res.status(500).json({ error: 'Error interno' });
        }
    }
);
```

**Vulnerabilidades corregidas:**
1. Inyección SQL (concatenación) → prepared statements
2. Exposición de datos (SELECT *) → solo columnas necesarias
3. No validación de entrada → express-validator
4. Error expuesto al cliente → logging + mensaje genérico
5. Sin cabeceras de seguridad → helmet
6. Sin rate limiting → express-rate-limit
7. Sin CORS configurado → cors con orígenes permitidos
8. No usa parámetro de ruta → :id en lugar de query string

## Resumen

- **OWASP Top 10**: guía de las vulnerabilidades más críticas
- **Inyección**: siempre prepared statements, validación y sanitización
- **Cifrado**: bcrypt para contraseñas, JWT con secret fuerte
- **Helmet**: cabeceras HTTP de seguridad en una línea
- **CORS**: control de acceso entre dominios
- **.env**: secretos fuera del código, nunca en Git
