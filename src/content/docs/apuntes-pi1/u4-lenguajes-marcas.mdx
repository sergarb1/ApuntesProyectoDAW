## Introducción

Los lenguajes de marcas utilizan etiquetas para estructurar y describir la información. HTML presenta contenido web, XML transporta datos con estructura personalizada y JSON es el formato ligero más usado en APIs modernas.

## HTML5 Semántico

HTML5 introduce etiquetas semánticas que describen el significado del contenido, mejorando la accesibilidad y el SEO.

```html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Proyecto</title>
</head>
<body>
    <header>
        <nav>
            <ul>
                <li><a href="/">Inicio</a></li>
                <li><a href="/about">Sobre nosotros</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <article>
            <header>
                <h1>Título del artículo</h1>
                <time datetime="2026-03-15">15 de marzo de 2026</time>
            </header>
            <p>Contenido del artículo...</p>
        </article>

        <aside>
            <h2>Barra lateral</h2>
            <p>Información relacionada</p>
        </aside>
    </main>

    <footer>
        <p>&copy; 2026 - Todos los derechos reservados</p>
    </footer>
</body>
</html>
```

### Etiquetas semánticas principales

| Etiqueta | Propósito |
|----------|-----------|
| `<header>` | Cabecera de página o sección |
| `<nav>` | Navegación principal |
| `<main>` | Contenido principal (único por página) |
| `<article>` | Contenido independiente y reutilizable |
| `<section>` | Agrupación temática de contenido |
| `<aside>` | Contenido complementario |
| `<footer>` | Pie de página o sección |
| `<figure>` | Contenido ilustrado (imagen, diagrama) |
| `<figcaption>` | Leyenda de figure |

## XML (eXtensible Markup Language)

XML es un lenguaje de marcas que define reglas para codificar documentos de forma legible por humanos y máquinas.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<proyecto>
    <titulo>Gestor de Tareas</titulo>
    <version>1.0</version>
    <equipo>
        <miembro rol="desarrollador">
            <nombre>Ana García</nombre>
            <email>ana@example.com</email>
        </miembro>
        <miembro rol="tester">
            <nombre>Luis Pérez</nombre>
            <email>luis@example.com</email>
        </miembro>
    </equipo>
    <tareas>
        <tarea estado="completada">
            <id>1</id>
            <descripcion>Diseñar base de datos</descripcion>
            <prioridad>alta</prioridad>
        </tarea>
        <tarea estado="en_progreso">
            <id>2</id>
            <descripcion>Implementar API REST</descripcion>
            <prioridad>alta</prioridad>
        </tarea>
    </tareas>
</proyecto>
```

### Reglas XML

1. **Un único elemento raíz**: todo el contenido dentro de una etiqueta principal
2. **Cierre obligatorio**: `<tarea></tarea>` o `<tarea />` (vacío)
3. **Diferenciación mayúsculas/minúsculas**: `<Tarea>` ≠ `<tarea>`
4. **Anidamiento correcto**: no mezclar aperturas/cierres
5. **Atributos entre comillas**: `estado="completada"`

## XSD (XML Schema Definition)

XSD define la estructura, tipos y restricciones de un documento XML.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
    <xs:element name="proyecto">
        <xs:complexType>
            <xs:sequence>
                <xs:element name="titulo" type="xs:string"/>
                <xs:element name="version" type="xs:string"/>
                <xs:element name="equipo" type="EquipoType"/>
                <xs:element name="tareas" type="TareasType"/>
            </xs:sequence>
        </xs:complexType>
    </xs:element>

    <xs:complexType name="EquipoType">
        <xs:sequence maxOccurs="unbounded">
            <xs:element name="miembro" type="MiembroType"/>
        </xs:sequence>
    </xs:complexType>

    <xs:complexType name="MiembroType">
        <xs:sequence>
            <xs:element name="nombre" type="xs:string"/>
            <xs:element name="email" type="xs:string"/>
        </xs:sequence>
        <xs:attribute name="rol" type="xs:string" use="required"/>
    </xs:complexType>

    <xs:complexType name="TareasType">
        <xs:sequence maxOccurs="unbounded">
            <xs:element name="tarea">
                <xs:complexType>
                    <xs:sequence>
                        <xs:element name="id" type="xs:int"/>
                        <xs:element name="descripcion" type="xs:string"/>
                        <xs:element name="prioridad" type="PrioridadType"/>
                    </xs:sequence>
                    <xs:attribute name="estado" type="EstadoType" use="required"/>
                </xs:complexType>
            </xs:element>
        </xs:sequence>
    </xs:complexType>

    <xs:simpleType name="PrioridadType">
        <xs:restriction base="xs:string">
            <xs:enumeration value="alta"/>
            <xs:enumeration value="media"/>
            <xs:enumeration value="baja"/>
        </xs:restriction>
    </xs:simpleType>

    <xs:simpleType name="EstadoType">
        <xs:restriction base="xs:string">
            <xs:enumeration value="completada"/>
            <xs:enumeration value="en_progreso"/>
            <xs:enumeration value="pendiente"/>
        </xs:restriction>
    </xs:simpleType>
</xs:schema>
```

## JSON (JavaScript Object Notation)

JSON es un formato ligero de intercambio de datos, derivado de JavaScript.

```json
{
    "proyecto": {
        "titulo": "Gestor de Tareas",
        "version": "1.0",
        "equipo": [
            {
                "nombre": "Ana García",
                "email": "ana@example.com",
                "rol": "desarrollador"
            },
            {
                "nombre": "Luis Pérez",
                "email": "luis@example.com",
                "rol": "tester"
            }
        ],
        "tareas": [
            {
                "id": 1,
                "descripcion": "Diseñar base de datos",
                "prioridad": "alta",
                "estado": "completada"
            },
            {
                "id": 2,
                "descripcion": "Implementar API REST",
                "prioridad": "alta",
                "estado": "en_progreso"
            }
        ]
    }
}
```

### Tipos de datos JSON

| Tipo | Ejemplo |
|------|---------|
| String | `"Hola mundo"` |
| Number | `42`, `3.14` |
| Boolean | `true`, `false` |
| Null | `null` |
| Array | `[1, 2, 3]` |
| Object | `{"clave": "valor"}` |

### JSON vs XML

| Característica | JSON | XML |
|---------------|------|-----|
| Legibilidad | Más compacto | Más verboso |
| Tipos de datos | Nativos (string, number...) | Solo texto |
| Metadatos | No tiene | Atributos, namespaces |
| Validación | JSON Schema | XSD, DTD |
| Uso típico | APIs REST, config | Documentos, SOAP |

## Transformaciones entre formatos

### JavaScript: XML ↔ JSON

```javascript
// Objeto a JSON
const proyecto = { titulo: "Mi App", version: "1.0" };
const json = JSON.stringify(proyecto, null, 2);

// JSON a objeto
const datos = JSON.parse(json);
console.log(datos.titulo);
```

### Java: XML ↔ JSON con Jackson

```java
// JSON a objeto
ObjectMapper mapper = new ObjectMapper();
Proyecto p = mapper.readValue(jsonString, Proyecto.class);

// Objeto a JSON
String json = mapper.writerWithDefaultPrettyPrinter()
    .writeValueAsString(proyecto);
```

## Buenas prácticas

1. **HTML semántico**: usa etiquetas con significado, no `<div>` para todo
2. **Accesibilidad**: añade `alt` en imágenes, `label` en formularios, `aria-*` donde necesario
3. **JSON para APIs**: es más ligero y rápido de parsear que XML
4. **Validar siempre**: usa XSD para XML y JSON Schema para JSON en producción
5. **Indentación consistente**: mejora la legibilidad de cualquier formato

## Resumen

- **HTML5**: estructura semántica para la web (header, main, article, footer)
- **XML**: formato extensible y validable mediante XSD
- **JSON**: formato ligero ideal para APIs y configuraciones
- La validación (XSD/JSON Schema) garantiza la integridad de los datos
- Elegir el formato adecuado depende del caso de uso
