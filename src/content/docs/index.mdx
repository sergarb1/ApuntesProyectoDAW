## 📋 PI1 — Proyecto Intermodular I

PI1 (1º DAW) — Transforma una idea en un **MVP funcional**.

<div class="card-grid">

<a href="/ApuntesProyectoDAW/pi1/fases/reto-1/" class="card-link">
<div class="card">

<div class="card-title">🔍 Reto 1 — Identificación</div>

Define el problema que resuelve tu proyecto. Crea un Lean Canvas, define el público objetivo y valida que la idea tenga sentido antes de escribir una línea de código.

<span class="card-btn">👉 Ver reto</span>
</div>
</a>

<a href="/ApuntesProyectoDAW/pi1/fases/reto-2/" class="card-link">
<div class="card">

<div class="card-title">📋 Reto 2 — Análisis</div>

Especifica qué debe hacer tu proyecto. Redacta requisitos funcionales, historias de usuario, casos de uso y define el alcance realista del MVP.

<span class="card-btn">👉 Ver reto</span>
</div>
</a>

<a href="/ApuntesProyectoDAW/pi1/fases/reto-3/" class="card-link">
<div class="card">

<div class="card-title">✏️ Reto 3 — Diseño</div>

Diseña la solución antes de programar. Diagramas UML, modelo entidad-relación, wireframes y prototipos que guíen la implementación.

<span class="card-btn">👉 Ver reto</span>
</div>
</a>

<a href="/ApuntesProyectoDAW/pi1/fases/reto-4/" class="card-link">
<div class="card">

<div class="card-title">⚙️ Reto 4 — Implementación</div>

Construye el MVP funcional. Configura la base de datos, desarrolla backend y frontend, versiona con Git y despliega una primera versión.

<span class="card-btn">👉 Ver reto</span>
</div>
</a>

<a href="/ApuntesProyectoDAW/pi1/fases/reto-5/" class="card-link">
<div class="card">

<div class="card-title">🎤 Reto 5 — Presentación</div>

Prepara la defensa final. Demo del producto funcional, memoria del proyecto, vídeo resumen y presentación oral. El cierre del PI1.

<span class="card-btn">👉 Ver reto</span>
</div>
</a>

</div>

## ⚡ PI2 — Proyecto Intermodular II

PI2 (2º DAW) — Convierte el MVP en un **producto profesional**.

<div class="card-grid">

<a href="/ApuntesProyectoDAW/pi2/fases/sprint-0/" class="card-link">
<div class="card">

<div class="card-title">🔎 Sprint 0 — Auditoría</div>

Evalúa el proyecto heredado de PI1. Revisa código, deuda técnica, documentación y define qué mantener, qué refactorizar y qué descartar.

<span class="card-btn">👉 Ver sprint</span>
</div>
</a>

<a href="/ApuntesProyectoDAW/pi2/fases/sprint-1/" class="card-link">
<div class="card">

<div class="card-title">🏗️ Sprint 1 — Arquitectura</div>

Refactoriza la base del proyecto. Aplica patrones de diseño, separa responsabilidades y prepara el código para escalar profesionalmente.

<span class="card-btn">👉 Ver sprint</span>
</div>
</a>

<a href="/ApuntesProyectoDAW/pi2/fases/sprint-2/" class="card-link">
<div class="card">

<div class="card-title">🎨 Sprint 2 — Frontend</div>

Construye una interfaz profesional con un framework moderno (React, Vue o Angular). Componentes, estado, enrutamiento y diseño responsive.

<span class="card-btn">👉 Ver sprint</span>
</div>
</a>

<a href="/ApuntesProyectoDAW/pi2/fases/sprint-3/" class="card-link">
<div class="card">

<div class="card-title">⚡ Sprint 3 — Backend</div>

Desarrolla un backend robusto con APIs REST documentadas. Autenticación, validación, manejo de errores y conexión con la base de datos.

<span class="card-btn">👉 Ver sprint</span>
</div>
</a>

<a href="/ApuntesProyectoDAW/pi2/fases/sprint-4/" class="card-link">
<div class="card">

<div class="card-title">🧪 Sprint 4 — Testing</div>

Automatiza las pruebas. Tests unitarios, de integración y de seguridad. Asegura que el producto funciona antes de desplegar.

<span class="card-btn">👉 Ver sprint</span>
</div>
</a>

<a href="/ApuntesProyectoDAW/pi2/fases/sprint-5/" class="card-link">
<div class="card">

<div class="card-title">🚀 Sprint 5 — Despliegue</div>

Dockeriza la aplicación, configuras CI/CD y despliega en producción. Automatiza el proceso de principio a fin.

<span class="card-btn">👉 Ver sprint</span>
</div>
</a>

<a href="/ApuntesProyectoDAW/pi2/fases/sprint-6/" class="card-link">
<div class="card">

<div class="card-title">🎤 Sprint 6 — Defensa</div>

Presentación profesional tipo PechaKucha, vídeo demostrativo, manual de usuario y despliegue. El cierre del ciclo.

<span class="card-btn">👉 Ver sprint</span>
</div>
</a>

</div>
