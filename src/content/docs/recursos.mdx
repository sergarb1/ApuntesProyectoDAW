## Cómo buscar ayuda técnica

### Busca en el orden correcto

1. **Documentación oficial** (la fuente más fiable)
2. **Stack Overflow** (problemas comunes ya resueltos)
3. **GitHub Issues** (bugs conocidos, soluciones de la comunidad)
4. **Tutoriales / blogs** (guías paso a paso)
5. **Pregunta a un compañero o al profesor** (cuando lleves 30 min atascado)

### Cómo preguntar bien

Cuando preguntes (en Stack Overflow, al profesor, a un compañero),
sé específico:

> ❌ "No me funciona el login"
> ✅ "Al enviar el formulario de login con email y contraseña válidos,
>    recibo un error 500 en la consola del servidor. El código relevante es..."

Incluye siempre:
- Qué esperabas que pasara
- Qué pasa realmente
- El código relevante (no todo el archivo)
- Los mensajes de error completos

## Referencias por tema

### Frontend
- MDN Web Docs (la biblia del desarrollo web)
- CSS-Tricks (guías visuales de CSS)
- React/Vue/Svelte (documentación oficial, muy bien hecha)

### Backend
- Express.js (guía oficial)
- Node.js (documentación de la API)
- Spring Boot (guía oficial, muy completa)

### Bases de datos
- PostgreSQL / MySQL (documentación oficial)
- SQL Tutorial (w3schools, para empezar)

### Git
- Pro Git (libro gratuito, completo)
- Oh Shit, Git!?! (cuando algo sale mal)

### Docker
- Docker Curriculum (tutorial interactivo)
- Documentación oficial de Docker

## El recurso más infravalorado: tu profesor

El profesor no es solo para evaluar. Es tu **recurso principal**.
Pregúntale:
- ¿Voy por buen camino?
- ¿Esta decisión técnica tiene sentido?
- ¿Cómo harías tú esto?
- ¿Qué priorizarías ahora?

Una consulta de 5 minutos al profesor puede ahorrarte 2 días de trabajo.
