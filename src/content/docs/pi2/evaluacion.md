## Sistema de Evaluación

La evaluación de PI2 sigue el modelo del CEED, adaptado al ciclo DAW presencial. Se estructura en cinco instrumentos de calificación.

### Instrumentos de Calificación

| Instrumento | Descripción | Peso | Nota Mínima |
|---|---|---|---|
| PI1 Previo | Nota obtenida en Proyecto Intermodular I | 10% | >= 5 |
| Documentación | Auditoría, arquitectura, manual de usuario, documentación API | 20% | >= 5 |
| Proyecto | Implementación completa: frontend, backend, tests, Docker, despliegue | 40% | >= 5 |
| Defensa | Presentación PechaKucha, vídeo, mesa redonda | 20% | >= 5 |
| Participación | Colaboración en clase, foros, coevaluación en defensas | 10% | No |

### Requisitos para Aprobar

1. Tener aprobado **PI1** (nota >= 5)
2. Obtener >= 5 en cada instrumento (excepto participación)
3. La nota final es la media ponderada de los instrumentos
4. Si algún instrumento no se supera, la calificación máxima será 4.9

### Criterios de Evaluación por Sprint

#### Sprint 0 (Auditoría)
- Evalúa el análisis crítico del proyecto heredado
- Se valora la honestidad y profundidad del informe

#### Sprint 1 (Arquitectura)
- Evalúa el diseño arquitectónico y la refactorización
- Se valora la aplicación correcta de patrones

#### Sprint 2 (Frontend)
- Evalúa la calidad del frontend con framework
- Se valora diseño responsive, componentes y usabilidad

#### Sprint 3 (Backend)
- Evalúa la API REST y su documentación
- Se valora seguridad, estructura y completitud

#### Sprint 4 (Testing)
- Evalúa la cobertura y calidad de los tests
- Se valora el análisis de seguridad

#### Sprint 5 (Despliegue)
- Evalúa Docker, CI/CD y despliegue cloud
- Se valora la automatización y disponibilidad

#### Sprint 6 (Defensa)
- Evalúa la presentación y el vídeo
- Se valora la capacidad de argumentación

### Convocatorias

- **Convocatoria ordinaria**: tras la entrega final (Sprint 6)
- **Convocatoria extraordinaria**: finales de junio, mismos instrumentos
- Si PI1 está pendiente, no se puede aprobar PI2 hasta superarlo
