## Objetivo del Módulo

**Proyecto Intermodular II** se cursa en 2º de DAW. El objetivo es **convertir el MVP desarrollado en PI1 en un producto profesional completo**, aplicando las técnicas y herramientas avanzadas aprendidas en los módulos de segundo curso.

### Enfoque

El trabajo se organiza en **7 sprints** siguiendo la metodología Scrum. Cada sprint tiene una duración aproximada de 2-3 semanas y produce un incremento del producto evaluable.

### Módulos Relacionados

| Módulo | Contribución al Proyecto |
|---|---|
| DWEC (Desarrollo Web en Entorno Cliente) | Frontend avanzado con JavaScript, frameworks, AJAX |
| DWES (Desarrollo Web en Entorno Servidor) | Backend profesional, APIs REST, seguridad |
| DIW (Diseño de Interfaces Web) | UI/UX, diseño responsive, accesibilidad |
| DAW (Despliegue de Aplicaciones Web) | Servidores, Docker, despliegue cloud |
| Empresa e Iniciativa Emprendedora | Modelo de negocio, plan de viabilidad |
| Proyecto Intermodular | Integración de todos los conocimientos |

### Resultados de Aprendizaje

Al finalizar PI2, el alumno será capaz de:

- Auditar y refactorizar un código base existente
- Diseñar una arquitectura profesional multicapa
- Desarrollar un frontend moderno con framework
- Implementar APIs REST seguras y documentadas
- Aplicar testing automatizado en todos los niveles
- Desplegar la aplicación en un entorno cloud
- Defender el proyecto con argumentación técnica
