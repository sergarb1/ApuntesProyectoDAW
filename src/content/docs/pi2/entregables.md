## Entregables del Módulo

A continuación se enumeran todos los entregables que deben presentarse a lo largo del curso de PI2, organizados por sprint.

## Repositorio GitHub

- Código fuente completo del proyecto
- README.md con descripción del proyecto, tecnologías e instrucciones
- `.gitignore` actualizado
- Licencia del proyecto
- Commits frecuentes con mensajes descriptivos
- Ramas: `main`, `develop`, `feature/*`, `hotfix/*`

## Documentación Técnica

- Informe de auditoría del proyecto PI1
- Diagrama de arquitectura actualizado
- Documentación de la API (OpenAPI / Swagger)
- Diagramas UML del sistema completo
- Modelo E/R actualizado y migraciones
- Decisiones técnicas justificadas (ADR)

## Manual de Usuario

- Instrucciones de instalación y configuración
- Guía de uso de la aplicación
- Capturas de pantalla ilustrativas
- Solución de problemas comunes

## Docker

- Dockerfile para frontend y backend
- docker-compose.yml con todos los servicios
- Variables de entorno documentadas
- Scripts de inicialización

## Despliegue Cloud

- Aplicación desplegada y accesible públicamente
- URL del frontend en producción
- URL de la API en producción
- Pipeline CI/CD operativo

## Presentación Final

- Presentación PechaKucha (20 diapositivas)
- Vídeo demostrativo (4 minutos sin audio)
- Memoria técnica actualizada
- Enlace al repositorio GitHub

## Tabla Resumen

| Sprint | Entregable Principal | Formato |
|---|---|---|
| Sprint 0 | Informe de auditoría y plan de evolución | Documento PDF/MD |

| Sprint 2 | Frontend con framework moderno | Código + Demo |
| Sprint 3 | API REST documentada | Código + Swagger |
| Sprint 4 | Tests y análisis de seguridad | Código + Informe |
| Sprint 5 | Docker + Despliegue cloud | Docker + URL |
| Sprint 6 | Presentación + Vídeo + Manual | PDF + MP4 |
