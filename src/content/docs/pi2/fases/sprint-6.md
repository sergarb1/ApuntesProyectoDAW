## Objetivo

Preparar y realizar la defensa profesional del proyecto ante el tribunal. La presentación sigue el formato **PechaKucha** (20 diapositivas x 20 segundos) e incluye un vídeo demostrativo del funcionamiento.

## Entregables

### 1. Presentación PechaKucha

20 diapositivas con la siguiente estructura recomendada:

| # | Diapositiva | Duración |
|---|---|---|
| 1 | Portada: nombre del proyecto y alumno | 20s |
| 2 | Problema identificado | 20s |
| 3 | Solución propuesta | 20s |
| 4 | Lean Canvas | 20s |
| 5 | Tecnologías utilizadas | 20s |
| 6 | Arquitectura del sistema | 20s |
| 7 | Modelo de datos | 20s |
| 8 | Frontend: diseño y componentes | 20s |
| 9 | Backend: API y servicios | 20s |
| 10 | Seguridad implementada | 20s |
| 11 | Testing y calidad | 20s |
| 12 | Docker y contenerización | 20s |
| 13 | Despliegue cloud | 20s |
| 14 | CI/CD | 20s |
| 15 | Funcionalidades principales | 20s |
| 16 | Mejoras respecto a PI1 | 20s |
| 17 | Dificultades y soluciones | 20s |
| 18 | Líneas de futuro | 20s |
| 19 | Conclusiones | 20s |
| 20 | Agradecimientos | 20s |

### 2. Vídeo Demostrativo

Vídeo de **4 minutos sin audio** que el alumno comenta en directo:

- Recorrido completo por la aplicación
- Funcionalidades clave en funcionamiento
- Demostración del despliegue en producción
- Flujo completo: registro → uso → resultado

### 3. Manual de Usuario

Documento dirigido al usuario final:

- Cómo acceder a la aplicación
- Funcionalidades disponibles
- Guía paso a paso de las tareas principales
- Solución de problemas frecuentes

## Criterios de Evaluación

- Exposición clara, fluida y sin lectura de notas
- Vídeo demostrativo que muestra todas las funcionalidades
- Capacidad de argumentación técnica
- Respuesta adecuada a preguntas del tribunal
- Participación activa en la mesa redonda
