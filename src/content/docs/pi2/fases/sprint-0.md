## Objetivo

Realizar una auditoría completa del proyecto entregado en PI1 para identificar su estado actual, carencias técnicas y oportunidades de mejora. Este sprint sienta las bases del plan de trabajo del curso.

## Entregables

### 1. Informe de Auditoría

Documento que analice:

- **Código fuente**: estructura, organización, calidad, deuda técnica
- **Base de datos**: normalización, rendimiento, seguridad
- **Frontend**: usabilidad, responsive, accesibilidad
- **Backend**: seguridad, rendimiento, escalabilidad
- **Documentación**: completitud y claridad de la memoria
- **Git**: historial de commits, estructura de ramas

### 2. Análisis DAFO del Proyecto

| Aspecto | Evaluación |
|---|---|
| **Debilidades** | Carencias técnicas, código mejorable, funcionalidades incompletas |
| **Amenazas** | Tecnologías obsoletas, escalabilidad limitada, deuda técnica |
| **Fortalezas** | Base de datos funcional, idea validada, arquitectura base |
| **Oportunidades** | Nuevas tecnologías, mejoras de UX, ampliación de funcionalidades |

### 3. Plan de Evolución

- Lista de mejoras priorizadas por impacto y esfuerzo
- Mapeo de mejoras a sprints del curso
- Estimación de esfuerzo global

## Criterios de Evaluación

- Auditoría completa y honesta del estado del proyecto
- Identificación clara de deuda técnica y áreas de mejora
- Plan de evolución realista y alineado con los sprints
- Propuesta de mejoras justificadas técnicamente
